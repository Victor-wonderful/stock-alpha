## 1. 요약 (Summary)

Stock-Alpha는 흩어진 주식 데이터(재무·밸류에이션·수급·매크로·시그널)를 하나의 데이터 스토어에 모아, **AI가 "이 종목 살만한가"를 근거 있는 한 장의 리포트로 합쳐주는** 한국 주식 리서치 서비스다. 네이버·증권사가 "데이터(재료)"를 준다면 Stock-Alpha는 "판단(요리)"을 준다. 모든 수치는 코드가 계산하고(환각 차단), 모든 셋업은 백테스트 게이트를 통과해야만 발행된다(증거 기반·법규 안전).

## 3. 배경 (Background)

- **무엇인가:** 증권사 리서치센터 수준의 **멀티팩터 퀀트 + 펀더멘털(밸류에이션) + AI 애널리스트 리포트 + 4스타일 시그널**을 한 앱에 담는다. RSI/MACD식 아마추어 차트 분석이 아니다.
- **왜 지금인가:** 공개·저가 데이터(DART·FRED·KIS·pykrx)로 증권사급 재료를 모을 수 있게 됐고, LLM으로 그 재료를 사람이 읽는 판단으로 바꿀 수 있게 됐다. 개인투자자는 "데이터는 넘치는데 판단이 없는" 고통을 겪는다.
- **확정된 방향 전환(중요):** 초기 설계의 (a) **타인 대상 자동매매 봇**과 (b) **다크 터미널 UI**는 **폐기**됐다. 현재 확정 정체성은 **"AI 리서치 애널리스트 + 포트폴리오 진단"** (정보 제공 / 유사투자자문 영역, 정보만 판매). UI는 **모바일 퍼스트·결론 우선·토스식 평이한 언어**로 간다.
- **경쟁 포지셔닝:** *"InvestingPro의 펀더멘털 신뢰(적정가·건강점수) + Thinkpool의 수급·타이밍을, 한국 시장에 맞춰 한 앱에."* 두 경쟁사가 비운 자리(펀더멘털 × 한국 수급)를 점유한다.

## 4. 목표 (Objective)

**목표:** 바쁜 개인투자자가 종목 하나를 **30초 만에 "살만한가" 판단**하게 만들고, 자기 계좌의 건강을 주기적으로 점검하게 만든다.

**왜 중요한가:** "판단"은 네이버·증권사에 없는 빈자리이자, 구독료를 낼 이유다. 재방문(리텐션)은 포트폴리오 진단이 만든다.

**핵심 결과 (Key Results) — 측정 가능 목표:**
- KR1: AI 종목 리포트가 5~10종목에서 "환각 없이 쓸만한" 품질로 출력된다. (Phase 0 증명)
- KR2: 발행되는 모든 시그널이 백테스트 게이트를 통과한다(미통과 0건 발행).
- KR3: 7개 메뉴 IA가 웹·모바일에 모두 구현되고 실데이터로 동작한다.
- KR4: 무료→유료 전환 깔때기(리포트 요약 1줄→전문 페이월)가 작동한다.
- KR5: 유사투자자문업 신고 + 면책 고지가 유료 발행 전에 완료된다.

## 5. 대상 (Market Segment)

**핵심 페르소나 — "직장인 개미"**
- 종목 5~15개 보유. 시간이 없다. "더 사? 팔아?"가 늘 고민.
- 해결하려는 일(Jobs):
  - 아침 30초: "오늘 장 분위기 + 내 종목에 무슨 일?"
  - 종목 꽂힘: "이거 살만한가?"를 빠르게 판단
  - 알림 받음: "뭐가 변했나? 언제 팔아야 하나?"
  - 주말: "내 계좌 전체가 괜찮은가?" (건강검진)

**제약:**
- 1인 개발 — 범위를 단계적으로 증명하며 확장.
- 법규: 개인 맞춤 매매지시·수익보장 금지. 시그널은 전 구독자 동일·일방향(유사투자자문).
- 시세는 EOD(장마감) 중심. 실시간(분봉·틱)은 후속.
- 트랙레코드가 짧다 → "검증 엔진(백테스트)"으로 신뢰를 대체.

## 7. 솔루션 (Solution)

### 7.1 정보구조(IA) — 7개 메뉴 (각 메뉴 = 수익 깔때기 한 단계)

| # | 메뉴 | 역할 | 무료/유료 |
|---|---|---|---|
| ① | 홈 | 매일 여는 습관(리텐션) | 무료＋ |
| ② | 추천 | 엔진 엄선 픽(핵심 페이월) ⭐간판 | 유료 |
| ③ | 스크리너 | 시그널 탐색(거르기) | 부분유료 |
| ④ | 종목 | 검색·리서치 허브 | 부분유료 |
| ⑤ | 시장 | 무료 훅(SEO·유입) | 무료 |
| ⑥ | 내 자산 | 보유·관심·포트폴리오 진단(락인) | 유료 |
| ⑦ | 성과 | 트랙레코드 투명 공개(전환 엔진) | 무료＋ |

- **②추천 = 매매 계획서 카드**: 종목 + 검증·국면 배지 + 진입/목표/손절/R:R/비중 + "왜 추천"(점수·검증·국면) + 진입가 알림. 필터 없는 큐레이션(4~5종).
- **④종목 상세 = 스노우플레이크 5축**: 밸류(적정가·upside) · 수급(외인·기관) · 모멘텀(팩터) · 성장(재무) · 안정성(리스크). + 건강점수 1~5 · ProTips · AI 한 줄 판정. **5축·판정 무료, 적정가 수치·레벨 유료.**

### 7.2 주요 기능 (Key Features) — 사용자 기능 ← 뒤의 엔진

| 기능 | 해결 고민 | 뒤의 엔진 |
|---|---|---|
| **A. AI 종목 리포트** ⭐심장 | "이거 살만해?" | 밸류·팩터·리스크·수급·시그널 + 뉴스 |
| **B. 포트폴리오 진단** ⭐재방문 | "내 계좌 괜찮아?" | 리스크·팩터노출·집중도 |
| C. 오늘의 브리핑(홈) | "오늘 뭘 봐야 해?" | 레짐·섹터·관심종목 |
| D. AI 픽 / 스크리너 | "뭘 후보로?" | 멀티팩터·밸류·시그널 |
| E. 워치리스트 + 알림 | "뭐 변했어? 언제 팔아?" | 시그널·급변·공시 |
| F. 시장 대시보드 | "장 분위기?" | 레짐·섹터·매크로 |

**분석 엔진(재료) 구성:**
1. 멀티팩터 — Value/Quality/Momentum/Growth/LowVol/Size → 섹터중립 z → `composite_alpha`
2. 펀더멘털/밸류에이션 — 재무 정규화 → DCF + 상대가치 → `upside_pct`
3. 수급 — 기관/외인 순매수·공매도·프로그램
4. 매크로/레짐 — 2축 분류(방향 × 추세/횡보) → 국면별 셋업 라우팅
5. 리스크 — 베타·팩터노출·VaR
6. **백테스트 게이트** — IC·Sharpe·MDD·승률·R:R·워크포워드 (진위 판정자, 미통과 발행 금지)

**시그널 = 3축 조합 (style × setup × session):**
- 스타일 4종: 스캘핑·데이·스윙·포지션 (타임프레임·ATR배수·보유기간 분기)
- 셋업: 주도주추세·과대낙폭반등·돌파·종가베팅 + 통계/패턴 셋업 확장
- 가격 산출: ATR·지지/저항 기반 진입/SL/TP/R:R/포지션사이징

### 7.3 기술 (Technology)

- **모노레포**: `apps/web`(Next.js 15·TS) / `apps/mobile`(Expo SDK56·TS) / `apps/engine`(Python 3.12) / `packages/db`(SQL 마이그레이션·공유 타입) / `supabase`.
- **단일 공유 스토어**: 클라우드 Supabase(Postgres). 엔진/워커만 service_role로 write, 웹·모바일은 anon/authenticated read. 공개 시장데이터는 anon read, 사용자 소유 테이블은 RLS(`user_id = auth.uid()`).
- **운영**: 일일 배치 워커가 morning(08:30)·daily(16:30·EOD)에 인제스트→팩터→백테스트 게이트→시그널→리포트 실행. 신선도 가드로 낡은 데이터 발행 차단.
- **환각 차단**: 모든 수치는 Python 계산값을 LLM에 주입, LLM은 서술만, `source_refs`로 DB 근거 추적.

### 7.4 가정 (Assumptions)

- 사용자는 "수익 보장"보다 "검증된 근거 + 정직한 트랙레코드"를 더 신뢰한다. ⚠️ 검증 필요
- 공개·저가 데이터만으로 증권사급 판단 품질이 나온다(유료 컨센서스 없이 MVP). ⚠️ Phase 0에서 증명
- 독점 노하우가 없어도 "백테스트 검증 엔진"이 신뢰를 대체할 수 있다.
- 수급(외인·기관)이 한국 시장에서 펀더멘털을 의미 있게 교정한다.

## 8. 릴리스 (Release)

각 단계는 **"무엇을 증명하는가"**를 목표로 둔다.

**Phase 0 — 핵심 증명 (지금)** · 증명: "AI 리포트 한 장이 진짜 쓸만한가"
- ANTHROPIC_API_KEY 연결 → AI 리포트 실가동(엔진 수치 기반)
- 데이터 파이프 정리(KIS 단일화) · 5~10종목 품질·환각 점검

**Phase 1 — 트레이더급으로** · 증명: "리포트·시그널이 프로 규율을 담는가"
- 리스크 규율 엔진(R:R 게이트≥2·변동성 포지션사이징·분할·트레일링·계좌보호)
- 레짐 적응 연결(국면별 셋업 필터·비중)
- 셋업 확장(눌림목·변동성돌파 등) + 각 백테스트
- AI 리포트 "트레이더 렌즈"(청산 시나리오·무효화 트리거) · 뉴스 API

**Phase 2 — 리텐션 + UI** · 증명: "매일/매주 돌아오는가"
- B 포트폴리오 진단 화면 + 엔진 연결 · C 오늘의 브리핑 홈
- UI 전면 재설계(모바일 퍼스트·토스 톤) · 워치리스트/알림 고도화

**Phase 3 — 사업화** · 증명: "돈을 내는가"
- 무료/유료 경계 + 구독 결제 · **유사투자자문업 신고** · 면책·카피 컴플라이언스

**Phase 4 — 확장 (선택)**
- 프로 트레이더 제휴 브랜딩 · 미국주식 확장(FMP)

## 부록 B. 법규 정합 (유사투자자문)

- 시그널은 **전 구독자 동일·일방향·비개인화** → 유사투자자문(신고) 영역.
- **금지**: 개인 맞춤 매매지시, 양방향 상담, 수익률 보장·손실보전, 자동매매 설정 결합.
- 모든 산출물에 **면책 고지 + 출처(`source_refs`)** 부착. 성과는 과거 기록으로만 표기.
