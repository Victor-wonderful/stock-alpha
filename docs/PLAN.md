# Stock-Alpha 개발 계획 — 전문가급 주식 리서치·시그널·자동매매 플랫폼

> 승인된 설계 문서. 원본 플랜: `~/.claude/plans/iridescent-purring-boole.md`

## Context

목표는 RSI/MACD 수준의 아마추어 차트 분석이 아니라, **증권사 리서치센터에 준하는 멀티팩터 퀀트 + 펀더멘털 + AI 애널리스트 리포트** 플랫폼이다. 데이터는 시세·뉴스만으로 부족하다 — **재무제표·밸류에이션·컨센서스·수급(기관/외인/공매도)·매크로**를 모두 데이터 스토어에 모아 알파의 근거로 쓴다. 그 위에서 (1) **종목 추천**(스크리너·팩터 랭킹·모델 포트폴리오), (2) **AI 애널리스트 북**(투자의견·목표주가 담은 증권사 양식 리포트), (3) **시그널**을 산출하고, (Phase 3) 본인 증권계좌에서 **비수탁** 자동매매까지 이어진다.

핵심 원칙: **모든 엔진이 하나의 데이터 스토어를 공유**. 분석·추천·리포트·자동매매는 별개 제품이 아니라 같은 파이프라인의 단계.

### 확정 사항
- 자산군: **주식 중심** (KRX 우선 → 미국 확장). 자동매매는 **Phase 3**.
- 스택: **Next.js(TS) + Python**, DB/Auth = Supabase(Postgres).
- 데이터: **공개·저가 우선**(DART·SEC EDGAR·FRED·ECOS·FMP·yfinance·KRX) → 검증 후 유료 컨센서스/대체데이터.
- 분석: **멀티팩터 퀀트 + 펀더멘털(밸류에이션) + AI 리서치 통합**.
- 애널리스트 북: **4종** — 종목 인뎁스 / 정기 마켓 / 모델 포트폴리오·추천주 / 맞춤형.
- 투자 스타일: **4종 1급 차원** — 스캘핑·데이트·스윙·포지션. 스타일이 타임프레임·팩터가중·진입/TP/SL·R:R·보유기간·포지션사이징을 분기.
- 실시간 인프라: 스캘핑/데이트용 틱·호가(KIS WS)를 Phase 1부터.
- 가격 산출: **기술적 구조 기반**(ATR·지지/저항·스윙·변동성), R단위 손익비·포지션사이징.

## 시그널 3축 모델 (style × setup × session)

시그널은 **3개 독립 축의 조합**으로 표현된다. 같은 종목도 (스윙×과대낙폭반등×정규장)와 (데이트×종가베팅×종가)가 별개 시그널.

**축1 — 스타일(보유기간)**

| 스타일 | 타임프레임 | 주 신호원 | 보유기간 | 진입/TP/SL |
|---|---|---|---|---|
| 스캘핑 | 틱~1m | 호가 불균형·체결강도·VWAP | 초~분 | 타이트 ATR(1m)·호가 레벨, R:R 작게·고회전 |
| 데이트레이딩 | 1~15m | 일중 모멘텀·VWAP·당일 고저·갭 | 당일 청산 | 당일 지지/저항·ATR(5m), 장마감 강제청산 |
| 스윙 | 1h~1d | 추세·패턴·수급·카탈리스트 | 수일~수주 | 스윙 고저·일봉 ATR·이평 |
| 포지션 | 1d~1w | 멀티팩터·펀더멘털·밸류·매크로 | 수주~수개월 | 밸류 upside·주봉 구조, 와이드 SL |

**축2 — 셋업(플레이북)**: factor_composite(멀티팩터 종합) / leader_trend(주도주 추세) / oversold_bounce(과대낙폭 반등) / breakout(돌파) / close_betting(종가베팅) / theme(테마주, 후속) / new_listing(신규주, 후속)

**축3 — 세션**: pre(프리장) / regular(정규장) / close(종가단일가) / after(애프터장)

> 1차 구현: 주도주추세·과대낙폭반등·돌파·종가베팅 (기존 시세/수급 데이터로 가능). 테마·신규주·프리/애프터장은 데이터(테마 분류·상장정보·시간외 시세) 확보 후. 시장 레짐 대응은 macro 엔진이 셋업 선택·가중에 반영.

## 분석 엔진

1. **멀티팩터** — Value/Quality/Momentum/Growth/LowVol/Size → 섹터중립 z-score → 합성 `composite_alpha`.
2. **펀더멘털/밸류에이션** — 재무 정규화 → 비율 → DCF + 상대가치(peer) → `upside_pct`. 어닝 서프라이즈·컨센서스.
3. **수급** — 기관/외인 순매수·공매도·대차·프로그램.
4. **매크로/레짐** — 금리·환율·신용스프레드 → 국면 분류 → 팩터 가중 조정.
5. **리스크** — 베타·팩터노출·VaR·상관.
6. **백테스트/검증(게이트)** — IC·IR·Sharpe·MDD·턴오버·워크포워드. 미통과 시그널 발행 금지.
7. **마이크로구조(스캘핑/데이트)** — 호가 불균형·체결강도·VWAP 이탈.

## 가격 산출 모듈 (`engine/signals/levels`)

입력: 스타일·ATR·지지/저항·스윙포인트·변동성.
출력: `entry_price`, `stop_loss`, `tp1/tp2/tp3`, `risk_reward`(R), `position_size_pct`(= `risk_per_trade_pct` ÷ 진입-손절 거리), `holding_horizon`, `valid_until`.

## 종목 추천 & 애널리스트 북

- **추천(스타일별 분기)**: 스크리너 / 모델 포트폴리오(리밸런싱) / 테마·섹터 픽 / 컨빅션·진입·목표·손절.
- **AI 애널리스트 북(RAG)**: 수치는 코드 계산값 주입(환각 차단), LLM은 서술. `source_refs`로 근거 추적. 4종 발행 → `docx`/`pdf`. 투자의견·목표주가 면책 자동 첨부.

## 마일스톤

**Phase 1 (MVP)**
- M1 기반: 모노레포·docker-compose·Supabase 스키마+RLS·Next.js Auth.
- M2 인제스트: 마스터·OHLCV·재무(DART/EDGAR)·수급(KRX)·매크로(FRED/ECOS)·뉴스 + 실시간 틱/호가(KIS WS).
- M3 펀더멘털/밸류에이션 → `valuations`.
- M4 팩터·수급·매크로·마이크로구조 → `factor_scores`.
- M5 시그널 + 스타일별 가격 산출(`signals/levels`).
- M6 백테스트/검증(스타일별) → 게이트.
- M7 웹: 스타일 선택·스크리너·랭킹·종목 상세·대시보드·알림·티어.
- M8 구독/결제.

**Phase 2 (애널리스트 북)**
- M9 리포트 파이프라인(RAG → PDF/DOCX).
- M10 4종 리포트 + 정기 발행 + 열람/다운로드 + 면책.
- M11 품질 게이트(사실성·보안·성능·E2E).

**Phase 3 (봇)** — 한투 OpenAPI, KMS, 인스턴스 격리·킬스위치·멱등 주문, 규제 검토 선행. 별도 플랜.

## 검증 (E2E)

- 데이터: engine 1회전 후 `financials/flows/macro/factor_scores/valuations` 적재 SQL 확인.
- 분석 정확성: 재무비율·DCF 외부 출처 대조, 팩터 IC 유의성.
- 시그널 게이트: 미통과 발행 차단, 스타일별 백테스트(승률·R:R).
- 스타일 가격: 동일 종목 4스타일 다른 진입/TP/SL, `position_size_pct`·`risk_reward` 정합, 데이트 `valid_until` 당일 마감.
- 실시간: 장중 `ticks`/`orderbook` 적재 + 마이크로구조 지표.
- 추천: 필터·랭킹·모델포트 가중합=1·리밸런싱.
- 리포트: PDF/DOCX 수치 = DB 값, 면책 첨부.
- 웹: 로그인→스크리너→종목상세→리포트 (Playwright E2E + Preview MCP).
- 티어: Free/Pro/Premium RLS·서버 필터.
