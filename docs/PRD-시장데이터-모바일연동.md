# PRD — 시장 데이터 정직화 & 모바일 앱 실데이터 연동

> 작성일: 2026-06-29 · 브랜치: `feat/mobile-app` · 상태: 1·2단계 완료, 3단계 보류

## 1. 요약 (Summary)

Stock-Alpha의 시장 데이터가 "실시간"인 것처럼 보였지만 실제로는 하루 두 번만 갱신되는 마감 데이터였고, 장 시작 전 배치가 직전 종가를 "오늘 값"으로 잘못 저장하는 버그까지 있었다. 이 문제를 바로잡고(가짜 당일 지수 차단 + 정직한 "장마감 기준" 라벨), 동시에 모바일 앱(Expo)의 **시장 화면**과 **종목 목록 화면**을 목업 데이터에서 실제 Supabase DB로 연결했다.

## 3. 배경 (Background)

- **무엇에 관한 건가:** Stock-Alpha는 퀀트 + 펀더멘털 + AI 리포트 + 4스타일 시그널을 제공하는 주식 리서치 플랫폼이다. 웹과 모바일이 같은 Supabase 백엔드를 공유한다.
- **왜 지금인가:** 사용자가 "시장 데이터가 실시간으로 안 되는 것 같다"고 관측했다. 점검 결과, (a) 구조상 실시간 갱신 경로가 없고, (b) 화면에 "실시간 갱신"이라 거짓 표기되어 있었으며, (c) 장 시작 전 배치가 금요일 종가를 월요일 "오늘 값"으로 복사 저장하는 버그가 있었다.
- **최근 가능해진 것:** 모바일 앱은 13개 화면 목업이 완성된 상태였고, 화면을 하나씩 실제 DB에 연결하는 작업이 진행 중이었다. 시장·종목 화면이 그 다음 차례였다.

---

## 4. 목표 (Objective)

**목표:** 사용자에게 보여주는 숫자가 "정직"하고, 모바일 앱이 목업이 아니라 실제 데이터로 동작하게 만든다.

**왜 중요한가:** 투자 판단에 쓰이는 데이터다. "오늘 보합 0%"처럼 보이는 가짜 표시는 신뢰를 무너뜨린다. 정직한 기준일 표기와 실데이터 연동은 제품 신뢰성의 기본이다.

**핵심 결과 (Key Results):**
- KR1: 장중에 "오늘"로 표시되던 가짜 지수 행(직전 종가 복제) **0건** — 종가 확정 전 미적재. ✅ 달성
- KR2: "실시간 갱신" 등 데이터 갱신 주기를 오해시키는 문구 **제거**, "장마감 기준"으로 대체. ✅ 달성
- KR3: 모바일 시장 화면 3개 카드(매크로·수급·섹터) 실데이터 연동. ✅ 달성
- KR4: 모바일 종목 목록 화면 실데이터 연동 + 판정 필터 동작. ✅ 달성
- KR5: 모든 변경에 대해 타입체크 통과 + 실DB 데이터 검증. ✅ 달성

## 5. 대상 (Market Segment)

- **누구를 위한 것인가:** Stock-Alpha의 개인 투자자 사용자. 비로그인(쇼윈도) 방문자 포함 — 시장/종목 데이터는 anon 읽기 허용 영역이다.
- **해결하려는 일(Job):** "지금 시장이 어떤지, 어떤 종목이 분석됐는지를 신뢰할 수 있는 숫자로 빠르게 보고 싶다."
- **제약:**
  - 시스템은 EOD(장마감) 배치 구조다. 장중 실시간 시세 경로가 없다.
  - 웹·모바일 클라이언트에는 KIS 등 시세 API 키가 없다(Supabase anon 키만). 따라서 프론트가 직접 실시간 시세를 못 가져온다.
  - 수급 데이터는 금액(억원)이 아니라 주수 기준이 많아, 투자자별(개인/외국인/기관) 금액 분리가 어렵다.
  - 관심종목·포트폴리오는 사용자 소유 데이터(RLS `user_id = auth.uid()`)라 **로그인 없이는 접근 불가**.

## 6. 가치 제안 (Value Proposition)

| 사용자의 고통 | 우리가 주는 것 |
|---|---|
| "오늘 숫자라는데 안 움직인다 → 고장인 줄 안다" | 정직한 "○○일 장마감 기준" 라벨로 오해 제거 |
| "실시간이라더니 아니다 → 속았다는 느낌" | 거짓 "실시간" 문구 제거, 갱신 주기 명시 |
| "앱은 예시 데이터라 못 믿겠다" | 시장·종목 화면을 실제 분석 결과로 채움 |
| "어떤 종목이 매수/관망인지 한눈에 안 보인다" | 날짜별 분석 + 판정 필터 + 추천 종목 배지 |

**경쟁 대비 차별점:** 수치는 코드가 계산하고 DB 근거를 추적하는 "환각 차단" 원칙. 데이터가 없거나 오래됐으면 숨기거나 기준일을 명시 — 가짜로 채우지 않는다.

## 7. 솔루션 (Solution)

### 7.1 UX / 플로우

- **웹 홈/시장:** 마켓 스트립과 헤더에서 "실시간 갱신" → "장마감 데이터 기준"으로 변경. 시장 페이지 지수 스트립에 "지수는 MM/DD 장마감 기준 · 장중 실시간 미반영" 캡션 추가.
- **모바일 시장 화면:** 레짐 히어로(기존) + 매크로 카드(4지표) + 수급 카드(외인+기관 5일 순매수 + 순유입/유출 섹터) + 섹터 로테이션(상대강도 막대). 데이터 출처·기준일을 카드마다 표기.
- **모바일 종목 화면:** 검색 바 + 판정 필터 칩(전체/매수/중립/관망, 실제 동작) + 날짜별 분석 리스트(종목명·코드·판정·점수·요약, 추천 종목 배지). 탭하면 종목 상세로 이동. 빈 상태 처리.

### 7.2 주요 기능 (Key Features)

**A. 시장 데이터 정직화 (엔진 + 웹)**
1. `timeutil.kr_session_closed()` — 한국 정규장 마감(15:40 버퍼)/주말 판정 가드.
2. KIS·네이버 지수 인제스트: 종가 확정 전엔 "오늘" 날짜 지수 행 미적재 → 직전 종가 복제 차단.
3. 기존 DB의 가짜 당일 행 정리(16:30 배치가 실종가로 자동 복구).
4. 웹 홈 헤더·시장 페이지 라벨 정직화.

**B. 모바일 시장 화면 연동**
1. `getMarketMacro()` — `macro` 테이블에서 미국채10Y·원/달러·VIX·WTI 최신값 + 전일대비.
2. `getMarketBoard()` — `sector_rotation`에서 섹터 모멘텀(상대강도) + flow 합산(외인+기관 5거래일 순매수, 억원). 1조 이상은 조 단위 축약, 기준일 표기.

**C. 모바일 종목 목록 화면 연동**
1. `getStockAnalyses()` — `reports`(indepth·published, '거래 부적합' 제외)를 거래일별 그룹화, 일자 내 점수 내림차순.
2. `recommendations` 교차참조로 "추천 종목" 배지.
3. 판정 필터(전체/매수/중립/관망) 실제 동작 + 빈 상태.

### 7.3 기술 (Technology)

- 엔진: Python 3.12 (인제스트·배치 워커). 가드는 `apps/engine/engine`.
- 웹: Next.js 15 (서버 컴포넌트가 Supabase 직접 읽기).
- 모바일: Expo SDK 56 · expo-router · TypeScript. 쿼리 레이어 `src/lib/queries.ts`가 웹 `lib/data.ts` 미러.
- DB: 클라우드 Supabase. anon 키로 공개 시장 데이터 읽기, service_role은 엔진/워커 전용.
- 검증: `tsc --noEmit` 통과 + psycopg로 실DB 데이터 직접 확인.

### 7.4 가정 (Assumptions)

- 실제 일일 배치는 다른 운영 머신(C:\Stock-Alpha)에서 돈다(이 레포의 워커는 06-18 이후 정지). 가드 코드는 그 머신에 배포돼야 효력이 생긴다. ⚠️ 검증 필요
- `sector_rotation`이 06-09에 멈춰 있다(엔진 미갱신). 수급/섹터는 "최신이지만 오래된" 데이터 — 기준일로 정직 표기 중. ⚠️ 엔진 갱신은 별도 과제
- 사용자는 "실시간"보다 "정직한 기준일"을 더 신뢰한다고 가정.
