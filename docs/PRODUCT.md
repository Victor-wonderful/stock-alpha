# Stock-Alpha 제품 정의 & 개발 로드맵 (PRODUCT.md)

> 확정 방향: **AI 리서치 애널리스트 + 포트폴리오 진단** (정보 제공 / 유사투자자문 / 정보만 판매)
> 함께 보기: `docs/STRATEGY.md`(트레이더 노하우·시그널 엔진), 메모리 `stock-alpha-product-strategy`

## 한 문장
> **"바쁜 개인투자자가 종목 하나를 30초 만에 '살만한가' 판단하도록, 흩어진 데이터를 AI가 한 장의 근거 있는 리포트로 합쳐주는 서비스."**
> 네이버·증권사는 **데이터(재료)**, 우리는 **판단(요리)** 을 준다.

---

## 1. 기능 지도 (엔진 → 사용자 기능)

| # | 기능 | 해결 고민 | 뒤의 엔진 | 상태 |
|---|---|---|---|---|
| **A** | **AI 종목 리포트** ⭐심장 | "이거 살만해?" | 밸류·팩터·리스크·수급·시그널+뉴스 | 엔진✅ / LLM키·뉴스 필요 |
| **B** | **포트폴리오 진단** ⭐재방문 | "내 계좌 괜찮아?" | 리스크·팩터노출·집중도 | 엔진✅ / 화면 필요 |
| C | 오늘의 브리핑(홈) | "오늘 뭘 봐야 해?" | 레짐·섹터·관심종목 | 데이터✅ / 화면 필요 |
| D | AI 픽 / 스크리너 | "뭘 후보로?" | 멀티팩터·밸류·시그널 | ✅ 재포장 |
| E | 워치리스트 + 알림 | "뭐 변했어? 언제 팔아?" | 시그널·급변 | 일부✅ |
| F | 시장 대시보드 | "장 분위기?" | 레짐·섹터·매크로 | ✅ 완료 |

→ **엔진(재료)은 거의 다 있음.** 부족: ① LLM 키 ② 뉴스 ③ A·B 사용자 화면.

## 2. 페르소나 & 일상 플로우
직장인 개미. 종목 5~15개. 시간 없음. "더 사? 팔아?"가 늘 고민.
```
아침(모바일 30초)  → [C] 시장 분위기 + 내 종목 변화
종목 꽂힘         → 검색 → [A] 리포트로 판단
알림            → [E] 시그널 → [A] 확인
주말            → [B] 계좌 건강검진
```
모든 경로가 **[A] 리포트**로 수렴, 재방문은 **[B] 진단**이 만든다.

## 3. 핵심 우선순위
1. **A. AI 리포트** = 심장(차별화·과금 지점)
2. **B. 포트폴리오 진단** = 재방문 엔진(네이버에 없음)
3. C 브리핑 = 습관/획득  4. D·E·F = 발견·알림·맥락(보조)

## 4. 무료 vs 유료 (= 핵심가치=과금지점)
| 무료(획득) | 유료 구독(핵심) |
|---|---|
| 브리핑 / 대시보드 / 리포트 요약 1줄 / 워치 3개 | **리포트 전문**(근거·적정가·뉴스·트레이더 청산시나리오) / **진단 무제한** / 알림·AI픽 |

## 5. 차별화 = 깊이 + 검증
- 네이버는 "그래서 어떻다"를 안 줌. 우리는 줌(A).
- **환각 차단**: 수치는 코드가 계산·주입, LLM은 서술만, 출처 추적.
- **트레이더 노하우**: 검증된 셋업 + 리스크 규율(STRATEGY.md). "프로의 규율을 시스템화".
- **검증 엔진(백테스트)** 이 진위 판정자 → 무보장·증거기반(법규 안전).

## 6. 개발 로드맵 (단계별)

각 단계는 **"무엇을 증명하는가"** 를 목표로 둔다(리스크 최소화).

### Phase 0 — 핵심 증명 (지금~2주)
> 증명: "AI 리포트 한 장이 진짜 쓸만한가"
- [ ] **ANTHROPIC_API_KEY** → A 리포트 실가동(엔진 수치 기반, 뉴스 없이 MVP) — `reports/generate.py` 완료, 키만
- [ ] **데이터 파이프 정리**(KIS 단일화, pykrx/naver 레거시화) — 방향 무관 안전작업
- [ ] 리포트 5~10종목 실제 출력 검토(품질·환각 점검)

### Phase 1 — 트레이더급으로 (2~6주) ← STRATEGY.md 핵심
> 증명: "리포트·시그널이 프로 규율을 담는가"
- [ ] **리스크 규율 엔진** [STRATEGY §2]: R:R 게이트(≥2)·변동성 포지션사이징·분할·트레일링·계좌보호
- [ ] **레짐 적응 연결** [§3]: 국면별 셋업 필터·비중
- [ ] **셋업 확장 1~2개**(눌림목·변동성돌파) + 백테스트 게이트 [§5]
- [ ] **AI 리포트 트레이더 렌즈** [§7]: 청산 시나리오·손익비·무효화 트리거
- [ ] **뉴스 API**(네이버) → 리포트에 "왜 움직였나"

### Phase 2 — 리텐션 + UI 전면 재설계 (6~10주)
> 증명: "매일/매주 돌아오는가"
- [ ] **B 포트폴리오 진단** 화면 + 엔진 연결 (집중도·베타·시나리오, 정보 톤)
- [ ] **C 오늘의 브리핑** 홈
- [ ] **UI 전면 재설계**: 모바일 퍼스트·결론 우선·리포트 카드·토스 톤. **다크 터미널 폐기**
- [ ] 워치리스트/알림 고도화

### Phase 3 — 사업화 (10주+)
> 증명: "돈을 내는가"
- [ ] 무료/유료 경계 + **구독 결제**(M8)
- [ ] **유사투자자문업 신고** (유료 구체적 시그널 발행 전 필수)
- [ ] 면책·카피 컴플라이언스 점검(`docs/COMPLIANCE.md` 권장)

### Phase 4 — 확장 (선택·장기)
- [ ] 프로 트레이더 제휴 → 독점 "OO 전략" 브랜딩
- [ ] RA 테스트베드(일임) — 자본·인력·트랙레코드 갖춰지면
- [ ] 미국주식 확장(FMP 등)
